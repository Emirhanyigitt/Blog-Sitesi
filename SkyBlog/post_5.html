Temel Terminal Komutları


Bu makalemde temel terminal komutlarını ve ne işe yaradıklarını anlatacağım.

Windows Terminal , powershell , cmd ve linux için Windows subsystem(wsl ) gibi ortamlara erişim adına yeni bir merkezi komut satırı uygulamasıdır

Files Navigating

ls komutu dizini listelemek için kullanılır.

mkdir denemeklasor

Bu komutu verdiğimizde bulunduğumuz klasöre denemeklasor adında bir klasör oluşacaktır

ls -l: ls komutuyla benzer bir komuttur. Buradaki “-l” seçeneği “long” kelimesinin kısaltılmışıdır. Yani terminale diyoruz ki listele ama izinler kullanıcı isimleri vs. bana her şeyi göster.
ls -force : ls komutuyla benzer bir komuttur ayrıca gizli dosyaları da görmemizi sağlar
pwd: Bu komut hangi dizinde (klasörde) olduğunuzu bulmak için kullanılır.
Bulunduğumuz klasöre gider : cd .
Bulunduğumuz klasörün bir üst klasörüne gider : cd ./..
Bir üst klasöre gider (Üstteki ile aynı işi yapar) : cd ..
Bir üst klasörün içerisindeki file1’e gider : cd ../file1

rm file : dosya sil
rm -f dir : dosyayı kaldırmaya zorla
rm -r dir : dosyayı silmeye zorla
rm -rf dir : dosyayı kaldır
rm -rf/ : her şeyi siler ve geri dönüşü yoktur.

cp file1 file2 :1. ve 2. dosyayı kopyalar

mv file1 file2 :1. ve 2. dosyayı yeniden isimlendir

touch file komut satırı üzerinden yeni dosyalar oluşturmanıza izin verir

cat -dosyaların içeriğini çıkarır.
cat ex.txt- ex.txt- her iki dosya içeriğini de çıkarır.
cat a.txt > b.txt dosyanın üzerine yazı ekleme.cat a.txt >> b.txt dosyanın sonuna

cp — bir kaynaktan bir hedef konuma dosya veya klasör kopyalamak için kullanılır.

cp orijinal.txt hedef.txt

Linux dizin kopyalama
Dizin kopyalamak için yine cp komutunu kullanacağız fakat bunu yapmak için -r parametresinden kullanmamız gerekecek. Komut yapımız aşağıdaki gibidir.

cp -r [kaynak dizin] [hedef dizin]

Cat Komutu ve Kullanımı
Cat, Unix temelli işletim sistemlerinde sıklıkla kullanılan, temelde 3 farklı işlev sunan komutlardan biri. Bu işlevler; bir metin dosyası içeriğinin görüntülenmesi, kopyaların birleştirilmesi (farklı metin dosyası içeriklerinin vb.) ve kopyalardan (farklı metin dosyası içeriklerinden) yeni bir metin içeriği oluşturulması.

cat [ — q ] [ -r ] [ — s ] [ — S ] [ — u ][ — Z ] [ — n [ — b ] ] [ — v [ — e ] [ — t ] ] [ — | Dosya … ]

Opsiyonlar dışında en temel kullanımı şu şekildedir.

cat [dosya-adi]
cat file : dosyanın içeriğini görüntüler
cat > file :dosya oluşturduğumuzda içine metin koymasını sağlar.
cat >> file : çıktıyı dosyaya eklemeyi sağlar.

Tail Komutu
Tail sadece son birkaç satırı görüntüleyip çıkmak yerine, satırları gösterir ve sonra dosyayı izlemeye başlar.

tail -file: dosyanın çıktı içeriğini gösterir

Temel kullanım

tail [OPTION]... [FILE]...,

Proccesses

Linux programlardan prosesler oluşturur ve bu prosesleri çalıştırır. Birçok program çalıştırıldığında sadece tek bir proses oluşturur ancak bazı programlar farklı amaçlar için birden fazla proses oluşturabilirler. Örneğin Google Chrome, açılan her bir tab için yeni bir proses oluşturur.

Permissions
Okuma izni : Dosyanın okuma izni varsa içeriği görülebilir, dizinin okuma izni varsa içerdiği dosyaların listesi alınabilir.
Yazma izni : Dosyanın yazma izni varsa dosyayı değiştirebilir veya silebilirsiniz. Dizine yazma izni verildiğinde dizin altındaki dosyalar yazılabilir veya silinebilir.
Çalıştırma izni : Dosyayı çalıştırma hakkını verir.

r — Okuma izni ( Read permission ) w — Yazma izni ( Write permission ) x — Çalıştırma izni ( Execute permission )
-chmod actol file: dosya izinlerini değiştir
chmod 777 :herkes için rewx
chmod 755 :sahip için rw grup için rx
u — user (dosya veya dizin sahibi, yaratıcısı) g — group (dosya veya dizinin ait olduğu grup) o — other (diğerleri, user ve group haricindekiler) a — all (ugo- user,group, other) (hepsi, herkes dosya ve dizinlere erişebilir.)
